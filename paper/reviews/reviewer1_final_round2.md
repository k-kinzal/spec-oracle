## 致命点と最小修正案

### 1. 問題設定の非自己完結性（§0.1）
**問題**: 「多層防御における仕様統制の困難性」と述べているが、UAD/fが扱う「層」の定義が§0では未定義である。読者は§2まで待たされ、問題設定の具体的意味が分からない。

**修正案**: §0.1の冒頭に以下を追加:
```
本稿における「層」とは、要求仕様・API仕様・型・契約・テスト・実装など、
異なる抽象度で記述される仕様アーティファクトの各レベルを指す。
各層は独自の記述言語と検証手法を持ち、共通の根空間（root space）Ω
への部分射影 proj_i によって対応づけられる（詳細は§2）。
```

### 2. join/meet意味論の論理的不整合（§2.3, §3.1）
**問題**: 
- §2.3で「状態集合順序では join=∪, meet=∩」と定義
- §3.1で「U0は状態集合順序でのjoin（上限）」と主張
- しかし§3.1の定義式 `U0 := ⋃ lifted(i)` は「和」であり、順序論の「上限」ではない
- 上限(LUB)は「全ての lifted(i) を含む最小集合」を指すが、和∪は「全要素の合併」であり、概念が混在している

**論理的矛盾**: 
- 定理「U0_least_upper_bound_iff」(§4.6)は包含順序⊆でのLUB性を証明
- 一方§3.1は「U0は和だから over-approximation」と述べる
- 和=LUBが成立するのは **包含順序において** のみであり、§2.3の「状態集合順序」との関係が未定義

**修正案**: §2.3を以下に置き換え:
```
本稿では **包含順序** (S ⊆ T) のみを使用する。
- この順序において、join（最小上界）は ∪ で与えられる
- この順序において、meet（最大下界）は ∩ で与えられる
- 「仕様強度順序」への言及は削除（混乱の元）
```

§3.1を以下に修正:
```
U0 := ⋃ lifted(i) は、包含順序⊆における lifted層の最小上界(LUB)である。
すなわち、全ての i について lifted(i) ⊆ U0 が成立し、
この性質を持つ最小の集合がU0である（定理U0_least_upper_bound_iff）。
```

### 3. Lean証明と本文主張の不一致（§4.1）
**問題**: 
- 本文§4.1は「層間伝播定理」として `lifted(j) ⊆ lifted(i)` の **十分条件** を与えていると読める
- しかしLean実装 `lifted_transfer` を確認すると、これは **同値** ではなく条件付き包含の一方向証明である
- 「仮定1,2のもとで包含が成立」は証明されているが、「この仮定が必要最小か」は未検証
- §5判断2で「同一点連結仮定が必要」と述べているが、反例（仮定なしでは包含が破れる例）が提示されていない

**修正案**: §4.1のタイトルを以下に変更:
```
### 4.1 層間伝播の十分条件（lifted包含の導出）
```

本文に以下を追加:
```
注意: 本定理は仮定1,2の **十分性** を示すが、必要性は未証明である。
すなわち、より弱い仮定でも包含が成立する可能性を排除していない。
同一点連結仮定の必要性を示すには、この仮定を外した反例モデルの
構成が必要だが、本稿では扱わない（今後の課題）。
```

### 4. RQ6の評価不十分（§1, §6.2）
- RQ6は「実OSSアーティファクト抽出パイプラインを再実行可能な形で運用できるか」
- §6.2は再実行可能性（source-lock付き）を示しているが、「運用できるか」の評価基準が不明
- 3件の成功例だけでは「運用可能性」の十分性が未評価

**修正案**: §6.2に以下を追加:
```
RQ6の「運用可能性」評価には、以下が必要だが本稿では部分対応に留まる:
- スケーラビリティ（大規模プロジェクトへの適用可能性）: 未評価
- 自動化度（人手介入なしでの実行可能性）: regex抽出は半自動
- メンテナンス性（ソース変更への追従コスト）: 未評価
したがってRQ6は「技術的実行可能性」は示されたが、
「運用可能性」の全面的実証には至っていない。
```

## 判定理由
1. **問題設定の循環定義**: 層の概念が未定義のまま問題を述べている（自己完結性欠如）
2. **順序論の混乱**: 状態集合順序と包含順序が混在し、join/meetの定義が論理的に不整合
3. **Lean証明とのギャップ**: 本文が「必要条件」のように読めるが、Leanは「十分条件」のみ証明

これらは **理論・意味論の一貫性** に直接関わる致命的問題であり、修正なしでは形式的正当性を主張できない。
