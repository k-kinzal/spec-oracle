## 致命点と最小修正案

### 1. RQ5/RQ6の定義と結論の乖離（致命的）
**問題**: 
- RQ5は「抽出適合の型付き分解（theory）」、RQ6は「実抽出パイプライン実行可能性（practice）」と定義されている（§1）
- しかし§6.2と§10結論で「RQ5は抽象関係Eの定式化」「RQ6はsource-lock付き再実行」と記述
- §6.2の「RQ5 (theory) の実抽出器適用は対象外」と「RQ6 (practice) は抽出正当性を対象外」の記述が、RQ5/RQ6の境界を逆転させている

**修正案**:
1. RQ5定義を「抽象関係 `E : Ω → β_i → Prop` の型付き分解（soundness/completeness分離）」に明記（意味保存証明への接続は限界として）
2. RQ6定義を「実抽出パイプラインの**技術的再実行可能性**（抽出正当性は評価対象外）」に明記
3. §10結論を上記定義に完全一致させ、「RQ5は定理4.3で抽象関係レベルの分解を定式化、RQ6は§6.2/§7.5でパイプライン実行可能性を確認（抽出器自体の正当性は未評価）」と整理

### 2. 再現手順の網羅性不足（重大）
**問題**:
- §7.5でオンライン/オフライン手順を記述しているが、**SHA256不一致時の対処手順**が「例外停止」だけで終わっている
- リポジトリ同梱snapshotとlockが不整合の場合（第三者実行時）の復元手順が欠落
- 追試者が不一致に遭遇した場合の「論文実行時と同一入力の確保方法」が不明確

**修正案**:
§7.5に以下を追加:
```
4. オフライン追試でSHA256不一致が発生した場合は、
   論文実行時と異なる入力であり再現性が保証されないことを意味する。
   この場合、**同梱snapshotとlockの一致状態を復元**して再実行する:
   - `git checkout paper/case-study/real_projects/snapshots/`
   - `git checkout paper/case-study/real_projects/external_validation_sources.lock.json`
   - 上記2ファイルを論文提出時点の状態に戻し、オフライン再実行
```
